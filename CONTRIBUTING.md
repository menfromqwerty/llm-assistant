# Contributing

Спасибо за интерес к проекту.

## Перед изменениями

1. Создайте Issue для крупной функции или изменения архитектуры.
2. Не добавляйте API-ключи, `.env`, пользовательские сессии, модели, GGUF и EXE.
3. Создавайте отдельную ветку для каждой задачи.

## Локальная проверка

```powershell
python -m pip install -r requirements.txt
python -m pip install -r requirements-dev.txt
python -m compileall -q llm_assistant
python -m pytest -q
bandit -r llm_assistant -ll
```

## Pull Request

В описании укажите:

- что изменено;
- зачем это нужно;
- как проверено;
- меняются ли форматы сессий или настройки безопасности.

Код должен сохранять совместимость с Python 3.11+ и не выполнять команды или код модели без явного подтверждения пользователя.
