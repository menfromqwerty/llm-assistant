# Security Policy

## Supported version

Исправления безопасности принимаются для последней опубликованной версии.

## Reporting a vulnerability

Не публикуйте сведения об уязвимости в открытом Issue до исправления.
Используйте GitHub Private Vulnerability Reporting, если оно включено в репозитории,
либо свяжитесь с владельцем репозитория через профиль [@hikkiway](https://github.com/hikkiway).

В сообщении укажите:

- версию и операционную систему;
- шаги воспроизведения;
- возможное влияние;
- минимальный демонстрационный пример без реальных секретов.

## Secrets

Никогда не прикладывайте реальные:

- API-ключи;
- токены GitHub/Tavily/OpenAI;
- пароли;
- `.env`;
- пользовательские сессии с закрытым кодом.

## Threat model

Приложение рассматривает как недоверенные:

- ZIP-архивы;
- веб-страницы;
- ответы LLM;
- загруженные проекты;
- пользовательские URL и аргументы локального сервера.

Сгенерированный моделью код не должен запускаться без ручной проверки.
