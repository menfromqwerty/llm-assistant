# Security Policy

## Supported version

Исправления безопасности принимаются для последней опубликованной версии.

## Reporting a vulnerability

Не публикуйте сведения об уязвимости в открытом Issue до исправления. Используйте GitHub Private Vulnerability Reporting, если оно включено, либо свяжитесь с владельцем репозитория через профиль [@hikkiway](https://github.com/hikkiway).

Укажите версию, операционную систему, шаги воспроизведения, возможное влияние и минимальный пример без реальных секретов.

## Secrets

Не прикладывайте реальные API-ключи, токены, пароли, `.env`, пользовательские сессии и закрытый исходный код.

## Local password protection

В v16 защита запуска опциональна. Если пароль не задан, приложение открывается без запроса. После установки пароля:

- пароль проверяется через memory-hard KDF `scrypt`;
- пароль и ключ AES не записываются на диск;
- сессии шифруются AES-256-GCM в файлы `.llms`;
- изменение зашифрованных данных обнаруживается при расшифровке;
- после пяти неверных попыток приложение закрывается.

Забытый пароль восстановить нельзя.

## Threat model and limitations

Защита предназначена для конфиденциальности сохранённых сессий при копировании или краже файлов. Она не является DRM и не делает EXE невзламываемым. Локальный злоумышленник с доступом к учётной записи Windows, памяти запущенного процесса, отладчику или возможностью менять бинарник может обходить интерфейсные проверки.

Не шифруются:

- исходные файлы открытых проектов;
- локальные модели;
- имена файлов сессий;
- данные, уже переданные стороннему API;
- весь диск или профиль Windows.

Для защиты устройства используйте отдельную учётную запись Windows, блокировку экрана, BitLocker и резервные копии.

Приложение рассматривает как недоверенные ZIP-архивы, веб-страницы, ответы LLM, загруженные проекты, пользовательские URL и аргументы локального сервера. Сгенерированный код нельзя запускать без проверки.
