# Security audit — v13

Дата проверки: 2026-06-15.

## Scope

Проверены:

- сетевые запросы;
- обработка URL;
- ZIP и файловый ввод;
- хранение сессий и секретов;
- запуск локальных процессов;
- передача недоверенного контента модели;
- зависимости и GitHub CI.

## Исправленные проблемы

### High

1. **Запуск `.bat/.cmd` через `shell=True`**  
   Заменён на явный запуск системного `cmd.exe` с `shell=False`.

### Medium

1. **Zip Slip и запись файлов вне временного каталога**  
   `ZipFile.extractall()` удалён. Каждый путь нормализуется и проверяется.

2. **Хранение API-ключей в открытых JSON-сессиях**  
   Ключи больше не сохраняются. Старые сессии очищаются при загрузке.

3. **SSRF через загрузчик веб-страниц**  
   Разрешены только публичные HTTP(S)-адреса. Loopback, private, link-local,
   multicast и reserved IP блокируются до запроса и после redirect.

4. **Prompt injection из загруженных страниц и файлов**  
   Системный промпт явно помечает этот контент как недоверенные данные.

### Low / reliability

- добавлены ограничения размера и количества файлов ZIP;
- бинарные файлы архивов не распаковываются;
- максимальный размер одного файла ограничен;
- пользовательский LLM URL проверяется;
- исправлена ошибка двойного уничтожения окна API-ключей;
- ошибки загрузки страниц больше не скрываются полностью.

## Static analysis

Команда:

```text
bandit -r llm_assistant -ll
```

Результат после исправлений:

```text
No issues identified (Medium/High severity)
```

Оставшиеся предупреждения уровня Low относятся преимущественно к намеренному
использованию `subprocess` для запуска LM Studio, Ollama, Jan и llama.cpp, а также
к обработчикам интерфейса. Эти действия выполняются по команде пользователя.

## Remaining risks

- сессии содержат код и переписку в локальном незашифрованном JSON;
- пользователь может самостоятельно выбрать исполняемый файл llama.cpp;
- модель может вернуть вредный или ошибочный код;
- сторонние поисковые API получают текст поискового запроса;
- собственный HTTP-сервер без TLS передаёт данные открытым текстом;
- оценка токенов приблизительная.

## Recommended repository settings

- включить GitHub Secret Scanning и Push Protection;
- включить Dependabot alerts и updates;
- запретить прямой push в основную ветку;
- требовать прохождения CI перед merge;
- использовать GitHub Private Vulnerability Reporting.
